# site/ — rendered artifacts

Self-contained HTML pages that present puzzledesk output. Each is a single file with
inlined CSS/JS (no build, no external assets), so it renders anywhere it is served with
a `text/html` content-type.

## The sample set

A gallery of **playable** crosswords, each demonstrating a different part of the system.
Every grid was generated by the engines and clued live by the model behind the clue port
— nothing is hand-placed. Start at **`index.html`**.

| Page | Feature it shows | How the grid was made |
|------|------------------|-----------------------|
| `index.html` | the gallery | — |
| `mini.html` | the core object: a **double word square** (D1–D9) — a 5×5 with no black cells, every letter checked twice, all ten words ≥ 90 and distinct | `mini 5 90 1` |
| `blocked.html` | **black cells from a count** (D12/D13) — a rotation-symmetric, connected, fully-checked 5×5 layout found by search from the number 4 alone, then filled | `generate 5 5 4 72 1` |
| `blocked-asym.html` | the **symmetry toggle** (D13) — the same count-driven 5×5 search with 180° symmetry switched off, so the blacks fall freeform instead of in mirrored pairs | `generate 5 5 4 72 1 --nonsymmetric` |
| `midi.html` | **length-capped large minis** (D24/D25) — a 10×10 that fills from the same 2–5-letter lists a mini uses, because a max-entry-length cap holds every run short | `generate 10 10 0 66 1 --max-len 5` |
| `midi-asym.html` | the **symmetry toggle** on a 10×10 — the same cap-driven search with symmetry off, giving the older, lopsided crossword look | `generate 10 10 0 66 1 --max-len 5 --nonsymmetric` |
| `field.html` | the **Gibbs layout sampler** (D27) — a 10×10 whose black-cell pattern is *sampled* from an energy field (density spring + anti-cluster + a guaranteed no-2×2-block rule) rather than searched | `generate 10 10 0 63 1 --max-len 5 --gibbs` |
| `midi-long.html` | the **length cap lifted past five** (D36) — a 9×9 with entries to length 6 (RENDER, APIARY, VENEER, GRATES), on an asymmetric Gibbs-sampled layout; longer words are a *vocabulary* axis, not the double-square order | `generate 9 9 0 60 1 --max-len 6 --gibbs --nonsymmetric` |
| `saturday-mini.html` | the **difficulty model** (D21/D22) — a 5×5 the solver model *proves* is a Saturday (8 hard gets under Saturday cluing, 1 under weekday) | hand-clued; grid from `mini … --hard 6 --gimme 88` |

The player is black-cell aware and derives everything — entry structure, numbering,
navigation — from the embedded puzzle data. Click a square and type; click again (or press
space) to switch direction; **Check** marks wrong squares, **Reveal** fills the answer key.
Answers are checked in the browser and never stored. Light/dark theme follows the system and
has a manual toggle.

## Regenerating

The pages are committed so they can be viewed without re-running anything, but they are fully
reproducible. Two steps, from the repo root:

```bash
uv run --extra clue python site/generate_puzzles.py   # grids + live clues -> site/puzzles.json
uv run python site/render_pages.py                    # puzzles.json -> the HTML pages + index
```

The **grids** are deterministic (pinned seeds/params in `generate_puzzles.py`); the **clue
text** is generated fresh by the model each run, so re-running keeps the same grids with new
clues. The clue step needs the `clue` extra and a key (`Config.clue_api_key_env`, default
`ANTHROPIC_API_KEY_TWO`; see D16/D17). `render_pages.py` needs neither — it only reads
`puzzles.json`.

## Viewing it through GitHub

GitHub shows HTML as *source*, not rendered. Two ways to see a page:

- **Now, zero config** — open it through a raw-HTML CDN (serves the committed file with
  the right content-type):
  `https://raw.githack.com/<owner>/<repo>/<branch>/site/index.html`
- **Stable URL** — enable GitHub Pages for the repo (Settings → Pages). Pages serves from
  a branch's root or `/docs`, so to publish under a clean URL either move these files there
  or point Pages at a branch whose root holds them.

Nothing here is part of the Python package or the build; these are presentation artifacts.
`generate_puzzles.py`/`render_pages.py` are the build tooling for them, not package code
(they are outside `src/` and the lint/type scope).
