# site/ — rendered artifacts

Self-contained HTML pages that present puzzledesk output. Each is a single file with
inlined CSS/JS (no build, no external assets), so it renders anywhere it is served with
a `text/html` content-type.

## saturday-mini.html

A playable 5×5 "Saturday" mini — the grid generated by the difficulty model
(`app.difficulty.solve_order`, D22/D23) and clued by hand — plus the solver-model proof
of *why* it is a Saturday: the same grid needs 8 hard gets under Saturday cluing but only
1 under weekday cluing (D21/D22's word-vs-clue decoupling, made visible). The grid is
`DUETO/OHARA/RUSES/IRENE/CASTS × DORIC/UHURA/EASES/TRENT/OASES`; `mini 5 60 3 --max 90
--hard 6 --gimme 88` generates equivalents.

## Viewing it through GitHub

GitHub shows HTML as *source*, not rendered. Two ways to see the page:

- **Now, zero config** — open it through a raw-HTML CDN (serves the committed file with
  the right content-type):
  `https://raw.githack.com/<owner>/<repo>/<branch>/site/saturday-mini.html`
- **Stable URL** — enable GitHub Pages for the repo (Settings → Pages). Pages serves from
  a branch's root or `/docs`, so to publish under a clean URL either move the file there
  or point Pages at a branch whose root holds it.

Nothing here is part of the Python package or the build; these are presentation
artifacts, committed so they can be viewed without re-running anything.
